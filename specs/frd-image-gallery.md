# FRD: Image Gallery

**Feature ID**: F-002
**Status**: Draft
**Priority**: P0 (Critical)
**Last Updated**: 2026-03-26

## Description

The Image Gallery is the primary view of the Assets Manager application. It displays all uploaded images in a responsive Bootstrap card grid (3 columns on medium+ screens). Each card shows the image as a thumbnail preview, the filename (truncated), file size in KB, and last-modified date. The gallery includes a client-side auto-refresh mechanism that polls the server at adaptive intervals to detect when new thumbnails have been generated by the background worker.

The gallery is the default landing page — the root URL `/` redirects to `/storage` which renders the gallery. When no images exist, a friendly "No images found" message is displayed with a prompt to upload.

## User Stories

### US-F002-001: Browse All Images

**As a** Content Manager
**I want to** see all uploaded images in a gallery view
**So that** I can browse my image collection

**Acceptance Criteria:**
- GIVEN images have been uploaded WHEN I navigate to `/storage` THEN I see a card grid with all images showing preview, name, size, and date
- GIVEN no images have been uploaded WHEN I navigate to `/storage` THEN I see "No images found. Upload your first image!"

### US-F002-002: Auto-Refresh for Thumbnails

**As a** Content Manager
**I want to** see thumbnails appear automatically after upload
**So that** I don't have to manually refresh the page

**Acceptance Criteria:**
- GIVEN I just uploaded an image WHEN the thumbnail is generated by the worker THEN the gallery refreshes and shows the thumbnail without manual page reload
- GIVEN I uploaded an image WHEN polling is active THEN the page checks for updates every 3 seconds initially, slowing to 6 seconds after 30 seconds of no changes

### US-F002-003: Navigate to Image Detail

**As a** Content Manager
**I want to** click on an image card to view its full details
**So that** I can see the full-size image and metadata

**Acceptance Criteria:**
- GIVEN I am on the gallery page WHEN I click the "View" button on a card THEN I am navigated to the image detail page at `/storage/view-page/{key}`

## Functional Requirements

### FR-F002-001: List All Storage Objects

- Input: HTTP GET to `/storage`
- Processing: Call `StorageService.listObjects()` which retrieves all objects from the storage backend. In S3 mode, uses `ListObjectsV2Request`. In local mode, uses `Files.walk(rootLocation, 1)`. For each object, construct an `S3StorageItem` with key, name, size, lastModified, uploadedAt, and URL.
- Output: Thymeleaf `list.html` template rendered with `objects` model attribute containing `List<S3StorageItem>`
- Error handling: In local mode, `IOException` is caught and empty list returned. In S3 mode, exceptions propagate uncaught.

### FR-F002-002: Client-Side Polling

- Input: Page load or upload completion
- Processing: JavaScript `setInterval` fetches `/storage` via `fetch()`, extracts the image container HTML, and compares with current content. If changes are detected, the DOM is updated. Polling frequency adapts: 3s during active uploads (tracked via `sessionStorage`), 6s otherwise, and aggressive polling stops after 30s of no changes.
- Output: Updated gallery content without full page reload
- Error handling: `fetch()` errors logged to console; polling continues.

## Non-Functional Requirements

### NFR-F002-001: Responsive Layout

Gallery uses Bootstrap `col-md-4` grid, rendering 3 cards per row on medium+ screens and stacking on smaller screens.

### NFR-F002-002: No Pagination

All objects are loaded and rendered in a single page. No pagination, infinite scroll, or lazy loading exists.

### Known Limitations

- O(n²) performance in `AwsS3Service.listObjects()` — calls `findAll()` for every S3 object to look up upload timestamps (assessment finding A3)
- No pagination — all objects loaded at once; will degrade with large numbers of files
- Polling is client-side only — no WebSocket or SSE for real-time updates
- S3 `listObjectsV2` limited to 1000 objects by default with no pagination token handling
