| Concept | Description |
|---------|-------------|

## рҹ’Ў What is Spec2Cloud?

> рҹ“– **Full documentation:** [Spec2Cloud on GitHub](https://github.com/EmeaAppGbb/spec2cloud) | [Brownfield Guide](https://github.com/EmeaAppGbb/spec2cloud/blob/vNext/docs/brownfield.md)

**Spec2Cloud** is a framework that bridges the gap between **application specifications** and **cloud-ready implementations**. It uses AI-powered agents and GitHub Copilot skills to:

1. рҹ”Қ **Assess** вҖ” Analyze your legacy codebase and generate comprehensive assessment documents
2. рҹ“Ӣ **Plan** вҖ” Create a detailed, step-by-step modernization plan based on the assessment
3. вҡҷпёҸ **Execute** вҖ” Automatically implement the modernization changes following the plan
4. вң… **Validate** вҖ” Ensure the modernized application meets the original specifications

## рҹ’Ў What is Spec-Driven Development?

**Spec-Driven Development (SDD)** is a methodology where **specifications are the single source of truth** throughout the software lifecycle. Unlike traditional approaches where specs become outdated quickly, SDD ensures:

- рҹ“„ **Specs are living documents** вҖ” They evolve with the code and are always up to date
- рҹ”— **Traceability** вҖ” Every code change maps back to a specification requirement
- рҹӨ– **AI-Assisted** вҖ” GitHub Copilot uses specs as context to generate accurate, aligned code
- рҹ”„ **Iterative** вҖ” Specs and code evolve together through continuous feedback loops

## рҹ“Ұ The Assets Manager App

The **Assets Manager** is a multi-module Spring Boot application for managing image assets вҖ” uploading, viewing, downloading, and generating thumbnails. It's the perfect candidate for both **Java version modernization** and **cloud migration**! рҹ–јпёҸ

|-----------|-----------|

### вңЁ Application Features

- рҹ“Ө **Image Upload** вҖ” Upload images to S3 storage with metadata tracking
- рҹ“Ӣ **Image Listing** вҖ” Browse all uploaded images with metadata
- рҹ‘ҒпёҸ **Image Viewing** вҖ” View images and their thumbnails in the browser
- рҹ“Ҙ **Image Download** вҖ” Download original files
- рҹ—‘пёҸ **Image Deletion** вҖ” Remove images and associated thumbnails
- рҹ–јпёҸ **Thumbnail Generation** вҖ” Asynchronous thumbnail creation via a worker process and RabbitMQ
- рҹ’ҫ **Local Dev Mode** вҖ” Profile-based local file storage for development

## рҹӣ пёҸ Modernization Steps

Follow these steps to modernize the Assets Manager application from Java 8 to Java 21 and migrate from AWS to Azure using Spec2Cloud! рҹҡҖ

---

### рҹ“Ң Step 1 вҖ” Initialize Spec2Cloud

Open a terminal in the repository root and run:

```bash
npx spec2cloud init
```

You will be prompted to select the setup scope. Choose **рҹҸ—пёҸ Brownfield** since we are modernizing an existing application.

![Spec2Cloud Initialization](img/shell-intall.png)

> рҹ“– **Need more details?** Check out the full [Spec2Cloud Brownfield Documentation](https://github.com/EmeaAppGbb/spec2cloud/blob/vNext/docs/brownfield.md) for an in-depth guide on brownfield modernization scenarios.

> в„№пёҸ **What happens here?** Spec2Cloud scaffolds the necessary configuration files and Copilot skills into your repository, preparing it for the spec-driven modernization workflow.

---

### рҹ“Ң Step 2 вҖ” Launch GitHub Copilot

Start GitHub Copilot in your terminal:

```bash
copilot
```

> рҹ’Ў **Optional:** Use the `--yolo` flag for a smoother lab experience вҖ” this reduces the number of confirmation prompts so you can focus on the process:
>
> ```bash
> copilot --yolo
> ```

---

### рҹ“Ң Step 3 вҖ” Run Modernization Assessment

Use the **Modernization-Assessment** Spec2Cloud skill in Copilot:

```
/modernization-assessment
```

![Modernization Assessment](img/step1-modernize-assess.png)

Then, ask Copilot to **analyze the codebase** and generate the application architecture, Functional Requirements Documents (FRDs), and Product Requirements Documents (PRDs):

```
Analyse the codebase and generate specs
```

![Code Analysis](img/step1-modernize-analize.png)

рҹҺҒ **What you get:** A rich set of documentation to help you deeply understand the application and its modernization needs! Copilot can also generate diagrams and architecture documentation based on the code analysis.

---

### рҹ“Ң Step 4 вҖ” Review the Assessment

рҹ“– **Read the output assessment documents** carefully and understand the modernization approach proposed for this application.

Key things to look for:
- рҹ”Қ Identified dependencies and breaking changes (e.g., `javax.*` вҶ’ `jakarta.*`)
- рҹ“Ӣ Proposed target framework and platform (Spring Boot 3.x on Azure)
- вҡ пёҸ Risk areas and mitigation strategies (AWS SDK removal, S3 вҶ’ Blob Storage)
- рҹ“Ҡ Effort estimation

> рҹ”§ **Customization:** You can tell Copilot to adjust the assessment based on your specific intent and requirements!

For this lab, we will proceed with the proposed modernization to **Java 21** and migration from **AWS to Azure** вҳҒпёҸ

### рҹ“Ң Step 5 вҖ” Generate the Modernization Plan

Ask Copilot to generate a detailed modernization plan using the **Modernization Planning** Spec2Cloud skill:

```
/modernization-planner
```

![Modernization Planning](img/step1-modernize-plan.png)

> рҹ“Ӣ **What you get:** A comprehensive, step-by-step implementation plan that breaks down the modernization into manageable, ordered tasks.

---

### рҹ“Ң Step 6 вҖ” Move from AWS to Azure

Ask Copilot to include the cloud migration in the modernization plan:

```
include in the modernization plan a move from AWS to Azure
```

![AWS to Azure](img/step1-modernize-aws-azure.png)

This step ensures the plan covers:
- вҳҒпёҸ **AWS S3 вҶ’ Azure Blob Storage** вҖ” Replace the AWS S3 SDK with the Azure Storage SDK
- рҹ“Ё **RabbitMQ вҶ’ Azure Service Bus** вҖ” Migrate message queue to a managed Azure service
- рҹ—ғпёҸ **PostgreSQL вҶ’ Azure Database for PostgreSQL** вҖ” Move to a managed database service
- рҹ”§ **Configuration updates** вҖ” Update properties files for Azure service endpoints

---

### рҹ“Ң Step 7 вҖ” Review & Approve the Plan

рҹ“– **Read the modernization plan** thoroughly and understand each step that will be taken to modernize the application.

- вңҸпёҸ You can ask Copilot to **modify the plan** based on your needs
- вң… You will need to **give your consent** before Copilot begins executing the plan
- рҹ”„ The plan is iterative вҖ” you maintain full control at every stage

Once approved, the modernization process begins! рҹҸ—пёҸ

Copilot will **iterate through each step** of the modernization plan and execute the changes automatically. You can:
